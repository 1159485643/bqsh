# 明星同款舆情风险复审 V3 - 应用身份直连表格版

## 本版核心变化

- 不再使用飞书 OAuth 登录。
- 不再检查 user_access_token。
- App ID / App Secret 已合并到“数据源设置”。
- 用户填写 App ID、App Secret、飞书表格链接、处理人姓名后即可使用。
- 后端通过 App ID / App Secret 换取 tenant_access_token。
- 读取、领取、动态同步、审核回写全部使用应用身份访问表格。

## 必须满足

飞书表格必须把该应用加入文档权限，并给应用可编辑权限，否则应用身份无法读写该表。

## 保留优化

- 基础数据流式加载。
- 动态同步 60 秒周期，有变更 30 秒后再同步。
- 动态同步只同步：处理人 / 审核状态 / 审核备注 / 审核时间。
- 审核回写约 700ms 触发，1.5 秒兜底检查。
- 单线程动态同步调度，带超时和失败退避。
- 保留原审核 UI、图片预览、处理人筛选、Excel 合并按钮、领取任务、不通过快捷原因、不通过原因不能为空校验。

## Cloudflare Pages

- Build output directory 填 public

- 线下文件模式下，顶部不再显示后台动态同步周期、下次倒计时等同步计时，只显示“线下模式：本地复审进度实时保存”。

## 本次修复：防止作业过程中数据回跳

问题原因：
- 用户审核后，前端状态先变更；
- 结果尚未成功写回飞书时，后台动态同步可能读到飞书旧值；
- 旧值覆盖本地状态，导致页面“回跳”。

解决方案：
- 新增 pendingReviewRows 本地待回写保护队列。
- 新增 activeWriteRows 正在回写保护队列。
- 审核操作会立即标记为本地 pending，动态同步不会覆盖这些行。
- 写回成功后只清除对应版本的 pending，避免旧请求清掉新编辑。
- 写回失败时 pending 保留，重新加入写回队列，页面继续以本地审核结果为准。
- 动态同步如果发现远端已与本地 pending 一致，则自动确认并解除保护。
- 手动全量同步 / 流式加载后，会重新叠加本地 pending，防止全量同步造成回跳。


## 本次复查：已做数据重复跳出修复

复查结论：
- 上版解决了“待回写时被旧远端覆盖”的问题；
- 但写接口返回成功后，代码立刻清掉 pending；
- 飞书写接口成功和读接口可见之间可能有短暂延迟；
- 下一轮动态同步可能读到旧值，导致已做过的数据又变回未复审并重复跳出。

本次修复：
- 写回成功后不再立刻清 pending；
- 改为标记“已提交，待远端确认”；
- 只有动态同步读到远端状态/原因已追上本地，才真正解除保护；
- 如果远端还没追上，页面继续以本地已审核结果为准，不会重复跳出；
- 全量/流式加载时先恢复本地进度，再叠加 pending，保证待确认数据优先；
- 流式加载 append 时不再每批重置到第一页，避免作业过程中页面回跳。

- 红色选区框只在真实图片区域内移动，不再跑到白边区域导致位置不准。

- 放大倍率改为 6 倍。
- 不再用“小图宽高 / 倍率”简单估算红框。

- 小红框表示的范围，就是大图 overlay 当前显示的范围。
